# Personal Skills Repository (github.com/kaelig/skills)

## Problem Frame

Kaelig needs a public GitHub repo to host custom Claude Code skills that are installable via the `skills` CLI (`npx skills add kaelig/skills`) and downloadable as `.skill` files (ZIP archives that Claude Desktop can open directly). The first skill is a company-evaluator for employer research. More skills will be added over time.

## Requirements

- R1. Repo structure follows the montagao/skills convention: each skill is a directory at the root containing `SKILL.md` (plus optional `references/`, `scripts/` subdirectories), with `.skill` ZIP files also at the root
- R2. Skills use flat names (e.g., `company-evaluator`, not `kaelig:company-evaluator`)
- R3. Install via `npx skills add kaelig/skills` works out of the box — the repo structure must be compatible with the `skills` CLI's directory scanning
- R4. `.skill` files (ZIP archives of each skill directory) are auto-generated by a GitHub Action on push to `main` and committed back to the repo
- R5. The GitHub Action should only regenerate `.skill` files for skills whose source files actually changed, and clean up `.skill` files for deleted skills
- R6. The first skill is `company-evaluator` — a structured employer evaluation report generator (full content already written)
- R7. README documents installation methods (CLI and `.skill` download) and lists available skills with one-line descriptions

## Success Criteria

- Running `npx skills add kaelig/skills` from any machine installs the available skills
- Downloading a `.skill` file from the repo and opening it in Claude Desktop installs the skill
- Pushing a new `skill-name/SKILL.md` to `main` automatically generates `skill-name.skill` without manual steps
- Deleting a skill directory and pushing removes the corresponding `.skill` file

## Scope Boundaries

- No custom "create skill" skill — rely on external tooling (compound-engineering's `writing-skills` or `npx skills init`)
- No plugin manifest (`.claude-plugin/plugin.json`) — this is a skills repo, not a plugin
- No MCP servers or agents — just skills
- No npm publishing — distribution is GitHub-only (CLI + `.skill` files)
- No versioning/releases initially — `main` is the source of truth

## Key Decisions

- **Flat root structure over `skills/` subdirectory**: montagao/skills puts skill directories at the root, not under `skills/`. The `skills` CLI scans for `SKILL.md` files recursively, so both work. Root is simpler and matches the reference repo.
- **Committed `.skill` files over release assets**: `.skill` files live in the repo alongside source directories. Users can download directly from GitHub's file browser without needing the Releases page.
- **GitHub Action commits back to repo**: The action generates `.skill` ZIPs and commits them. This means the action needs write permissions and should use `[skip ci]` in its commit message to avoid infinite loops.
- **No namespace prefix**: For a personal repo with <20 skills, flat names are fine. Collision risk is negligible.

## Dependencies / Assumptions

- The `skills` CLI (`npx skills`) can install from `kaelig/skills` GitHub shorthand
- `.skill` files are standard ZIP archives containing `{skill-name}/SKILL.md` (and any reference files)
- GitHub Actions can commit back to the repo (requires appropriate permissions on the default `GITHUB_TOKEN` or a PAT)

# Skills Repository

Personal agent skills hosted at github.com/kaelig/skills.

## Structure

Each skill is a directory at the repo root containing `SKILL.md`:

```
skill-name/
  SKILL.md           # Required: skill definition with YAML frontmatter
  references/        # Optional: supporting docs loaded on demand
  scripts/           # Optional: helper scripts
```

`.skill` files (ZIP archives) are auto-generated by GitHub Actions — never create or edit them manually.

## Adding a new skill

1. Create `skill-name/SKILL.md` with frontmatter:
   ```yaml
   ---
   name: skill-name
   description: When to trigger and what it does
   ---
   ```
2. Add a row to the skills table in `README.md`
3. Commit and push to `main` — the GitHub Action generates `skill-name.skill` automatically

## Removing a skill

1. Delete the `skill-name/` directory
2. Remove its row from the `README.md` table
3. Push to `main` — the Action removes the orphaned `.skill` file

## Conventions

- Flat names, no namespace prefix (e.g., `company-evaluator`, not `kaelig:company-evaluator`)
- Skill directories live at the repo root, not under a `skills/` subdirectory
- Do not add `.skill` files to `.gitignore` — they are committed artifacts
- The `docs/` directory contains planning artifacts and is not part of the distributed skills
